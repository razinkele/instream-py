# v0.34.0 Per-reach PSPC output specification

**Date**: 2026-04-20
**Scope**: Arc K of the WGBAST-driven improvements roadmap
**Plan**: `docs/superpowers/plans/2026-04-20-arc-K-to-Q-wgbast-roadmap.md`

## Purpose

WGBAST's canonical management metric is the **% of Potential Smolt
Production Capacity (PSPC)** achieved per wild salmon river. v0.34.0
emits this metric directly from SalmoPy so runs can be compared
cell-for-cell with the ICES Baltic salmon assessment.

Reference: ICES (2025) sal.27.22-31 + sal.27.32 stock annex, §3 —
"Management aim: achieve 75% of river-specific PSPC by 2020 for all
wild Baltic salmon populations."

## Output artifact

At end-of-run, when at least one reach has `pspc_smolts_per_year` set
in the config, SalmoPy writes `smolt_production_by_reach_{year}.csv`
to the output directory.

Schema:

| Column | Type | Description |
|--------|------|-------------|
| `year` | int | Calendar year (derived from `model.time_manager.current_date.year` at end-of-run) |
| `reach_idx` | int | Position in `config.reaches` dict (0-indexed) |
| `reach_name` | str | Reach key from `config.reaches` |
| `smolts_produced` | int | Count of outmigrant records with `natal_reach_idx == reach_idx` |
| `pspc_smolts_per_year` | float or blank | Configured PSPC; blank (→ NaN) if unset |
| `pspc_achieved_pct` | float or blank | `100 × smolts_produced / pspc`; blank if PSPC unset or ≤ 0 |

Reaches without configured PSPC still emit a row reporting
`smolts_produced`, making the output usable for downstream analytics
even without an official PSPC envelope.

## Companion artifact: widened `outmigrants.csv`

`outmigrants.csv` also widens from 3 to 10 columns matching NetLogo
InSALMO 7.3's `Outmigrants-<run>.csv`:

```
species, timestep, reach_idx, natal_reach_idx, natal_reach_name,
age_years, length_category, length_cm, initial_length_cm, superind_rep
```

The `natal_reach_idx` column is the per-river key driving the PSPC
rollup. `natal_reach_name` is carried in parallel for join convenience.

## PSPC values for WGBAST assessment rivers

Order-of-magnitude values from WGBAST 2026 §3 river-specific tables:

| River | AU | PSPC (smolts/yr) | Notes |
|-------|----|------------------|-------|
| Tornionjoki | 1 | ~2,200,000 | Largest Main-Basin stock |
| Simojoki | 1 | ~95,000 | Trap-counted monitoring river |
| Byskeälven | 2 | ~180,000 | Skoglund 2024 homing dataset |
| Mörrumsån | S | ~50,000 | Southern Baltic, 1-2yr smolts |

These are canonical references. Arc M (multi-river Baltic fixtures)
will add per-river configs keyed to these values.

## Baltic case study placeholders

The Baltic fixture (`configs/example_baltic.yaml`, 10 reaches, Nemunas
basin + Curonian Lagoon + Baltic coast) uses *preliminary* PSPC values
on the 3 primary anadromous reaches:

| Reach | PSPC (smolts/yr) | Justification |
|-------|------------------|---------------|
| Nemunas | 5,000 | main channel lower 40 km |
| Atmata | 1,500 | anadromous-corridor share |
| Minija | 1,200 | key Lithuanian spawning tributary |

These are placeholders pending a Kesminas et al. Nemunas-basin
literature review. Total basin ≈ 8,000 smolts/yr is consistent with
ICES sal.27.32 Gulf of Finland recruitment order-of-magnitude.

Non-natal reaches (`CuronianLagoon`, `BalticCoast`) correctly leave
`pspc_smolts_per_year` unset — smolts are counted by their birth reach,
not their exit reach.

## Caveats

### Super-individual rep-weight

v0.34.0 counts outmigrant records (= super-individuals), not
rep-weighted smolt equivalents. For configs with `superind_max_rep > 1`,
divide the WGBAST comparable smolt count by the modal rep-weight to
get back to individual smolts, or extend the function to sum
`superind_rep` when this becomes a blocker.

### Single-year window

`write_outputs` fires once at end-of-run, so the CSV captures the
*last* simulation year's production. For multi-year comparisons,
aggregate via the calibration framework's `aggregate_trajectories`
over multiple runs, or extend the hook to fire per year.

### Natal-reach overwrite at smoltification

`migrate_fish_downstream` overwrites `trout_state.natal_reach_idx[i]`
with `current_reach` at the SMOLT transition (an inherited v0.30.x
quirk). The outmigrant record is built *before* this overwrite, so the
CSV correctly captures the birth reach. Downstream uses of
`natal_reach_idx` on marine-stage fish (e.g., adult return) would see
the overwritten value — this is out of Arc K's scope but noted for
Arc O (straying/homing).
