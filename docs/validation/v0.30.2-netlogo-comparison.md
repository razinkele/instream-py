# v0.30.2 Python ↔ NetLogo InSALMO Run-Level Parity Report

**Date:** 2026-04-19
**inSTREAM-py version:** 0.30.2
**NetLogo reference:** InSALMO 7.3 (NetLogo 6.4.0) + InSALMO 7.4 (NetLogo 7.0.3), both byte-identical on example_a metrics
**Reference seed:** 98 (single seed only — no ensemble CI yet)
**Test fixture:** `example_a` (Chinook spring+fall, single-reach, 2011-04-01 → 2013-09-30)
**Test location:** `tests/test_run_level_parity.py::TestExampleARunVsNetLogo`

## Motivation

Until this report, NetLogo parity coverage for inSTREAM-py was
*function-level only*: 17 deterministic-function tests in
`tests/test_validation.py` (CMax, CStepMax, growth, survival-5-sources,
spawn suitability, fitness). No whole-model run-level comparison had
ever been executed against the cached BehaviorSpace tables sitting in
`netlogo-models/`, despite both codebases covering the same 2.5-year
example_a span.

The Sprint-4 deliverable in `docs/NETLOGO_PARITY_ROADMAP.md` called for
"behavioral checks beyond unit tests". Arc C of today's v0.30.2 plan
closes it.

## Methodology

- **Reference data (cached, unused until now):** `netlogo-models/insalmo_exampleA_results.csv`
  — 3,658 sub-daily ticks, single BehaviorSpace run, seed=98.
- **Python run:** `tests/test_behavioral_validation.py::_run_example_a()`
  pattern, instrumented per-day to track juvenile / adult counts +
  mean juvenile length + outmigrants, since the built-in census only
  emits twice yearly.
- **Metric choice:** five scalar statistics that are stable under the
  PCG64 ↔ Mersenne Twister RNG divergence. Per-individual trajectory
  matching is impossible with different RNGs; distributional summaries
  are the only meaningful comparison.

## Metrics & tolerances

| # | Metric | NetLogo target | Tolerance | Rationale |
|---|---|---|---|---|
| 1 | CH-S juvenile abundance peak | 2,151 on 2013-01-07 | rtol 0.30 | Peak smooths RNG noise, 30% tolerance ≈ 1-sigma at this N |
| 2 | CH-S adult abundance peak | 21 on 2012-07-09 | atol 8 | Small-N metric, absolute tolerance appropriate |
| 3 | CH-S small outmigrant cumulative | 41,146 | rtol 0.20 | Sum-based, most RNG-robust |
| 4 | CH-S juvenile mean length @ 2012-09-30 | 6.23 cm | rtol 0.10 | Intensive quantity (mean), tighter tolerance |
| 5 | CH-S outmigrant median date (50% CDF) | 2013-01-05 | ±14 days | Phenological, 2-week window covers drift from RNG |

Tolerances are deliberately loose — only one NetLogo seed is cached.
Tighten once a multi-seed BehaviorSpace ensemble is available.

## Known spots to check if parity fails

From `docs/NETLOGO_PARITY_ROADMAP.md` + `docs/calibration-notes.md`:

1. **`spawn_defense_area` semantic drift** (calibration-notes.md line 632):
   Python treats the value as Euclidean cm; NetLogo treats as area
   (cm²). example_a sets `spawn_defense_area: 200000` so interpretation
   matters.
2. **`light_turbid_const` offset** (NETLOGO_PARITY_ROADMAP line 138):
   a small per-reach constant that shifts light attenuation.
3. **Outmigration thresholds**: Python uses `outmigration_min_length=8.0`
   default (`io/config.py:321`), NetLogo uses 5/7 cm size classes per
   `parameters-ExampleA.nls:61`. If the outmigrant-total metric fails
   but peak juvenile matches, this is almost certainly the cause.

## Open gaps (follow-up work)

- **Single NetLogo seed only.** Re-run BehaviorSpace with 10–20 seeds
  and build distributional CIs; current tolerances are guesses.
- **Example B not cached.** Only example_a has NetLogo census. Adding
  example_b would cover multi-reach coordination.
- **InSALMO 7.4 requires NetLogo 7.x** which is not installed locally
  (only 6.4.0). The cached 7.4 CSV proves 7.3↔7.4 are byte-identical
  for example_a; no drift concern until 7.x is reinstalled.
- **Baltic case study has no NetLogo counterpart** — the new real-data
  reaches (Atmata / Minija / etc.) are Python-only. Parity claims for
  Baltic require either porting NetLogo (out of scope) or accepting
  that parity is function-level + Baltic-specific ecological plausibility.
