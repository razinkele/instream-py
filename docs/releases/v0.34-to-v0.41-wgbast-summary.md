## TL;DR

SalmoPy now produces outputs and accepts forcings directly comparable
to the ICES Working Group on Baltic Salmon and Sea Trout (WGBAST)
assessment:

- **Per-reach % PSPC achieved** — WGBAST's canonical management metric
- **Year-indexed M74 YSFM** forcing keyed by (year, river)
- **4 WGBAST-assessment-ready Baltic river fixtures** (Torne, Simo, Byske, Mörrum)
- **Per-cohort post-smolt survival forcing** keyed by smolt-year, stock-unit
- **Stray-fraction knob + spawner-origin MSA matrix** (WGBAST genetic MSA shape)
- **HELCOM grey-seal abundance scaling** (Holling Type II saturating)
- **Bayesian SMC posterior inference** (analogous to WGBAST's own Bayesian framework)

All opt-in knobs default to None/0.0 — **any run that doesn't opt in
behaves identically to v0.33.0 baseline and preserves NetLogo InSALMO
7.3 parity.**

## What ships in each release

### v0.34.0 — Arc K (Per-reach PSPC output)
End-of-run writes `smolt_production_by_reach_{year}.csv` (year,
reach_idx, reach_name, smolts_produced, pspc_smolts_per_year,
pspc_achieved_pct). `outmigrants.csv` widens from 3 to 10 NetLogo-
compat columns. New: `TroutState.initial_length`,
`TimeManager.start_date`, `ReachConfig.pspc_smolts_per_year`.

### v0.35.0 — Arc L (WGBAST M74 year-effect)
`redd_emergence` accepts `m74_forcing_csv`, `current_year`,
`river_name_by_reach_idx` kwargs. Applies binomial YSFM cull at
egg→fry emergence keyed by (year, river). Ships
`data/wgbast/m74_ysfm_series.csv`. Added `ReachConfig.river_name` +
`ReachParams.river_name`.

### v0.36.0 — Arc M (Multi-river Baltic fixtures)
4 new configs + fixtures: `example_tornionjoki.yaml` (PSPC 2.2M,
65.85°N, smolt_min 14cm), `example_simojoki.yaml` (95k, 65.60°N,
14cm), `example_byskealven.yaml` (180k, 64.98°N, 13cm),
`example_morrumsan.yaml` (60k, 56.17°N, 11cm). Generated via
`scripts/_scaffold_wgbast_rivers.py` + `scripts/_generate_wgbast_configs.py`.

### v0.37.0 — Arc N (Post-smolt survival forcing)
`marine_survival(current_year=)` overrides `background_hazard` for
fish in the post-smolt window keyed by SMOLT year (not calendar year).
Ships `data/wgbast/post_smolt_survival_baltic.csv`. New:
`MarineConfig.post_smolt_survival_forcing_csv`, `MarineConfig.stock_unit`.
Mathematical core: `daily_hazard = 1 - S_annual^(1/365)`.

### v0.38.0 — Arc O (Straying + MSA matrix)
- **Bug fix**: removed `natal_reach_idx = current_reach` overwrite at
  SMOLT transition (destroyed birth-reach signal).
- **Feature**: `MarineConfig.stray_fraction` at adult return;
  natal_reach_idx preserved, reach_idx reassigned on stray.
- **Output**: `write_spawner_origin_matrix` emits natal × spawning
  matrix (WGBAST MSA shape).

### v0.39.0 — Arc P (HELCOM grey-seal Holling II)
`seal_hazard(current_year=)` scales length-logistic base by Holling
Type II multiplier anchored at `seal_reference_abundance`. At r=1
(reference): mult=1.0. At r→∞: mult→k+1=3.0 (default k_half=2.0).
Linear scaling would have driven marine salmon extinct at 2021's 15×
abundance. Ships `data/helcom/grey_seal_abundance_baltic.csv`.

### v0.40.0 — Arc Q (Bayesian life-cycle wrapper)
New `instream.bayesian` subpackage: `Prior` dataclass,
`BALTIC_SALMON_PRIORS`, Poisson smolt-trap + negative-binomial
spawner-counter likelihoods, ABC-SMC sampler with tempered
log-likelihood. 8 tests including posterior-recovery on Gaussian-noise
toy problem. Ships `data/wgbast/observations/smolt_trap_counts.csv`.

### v0.41.0 — Arc 0 (Data-quality pass)
Literature-traced CSV upgrades via scite cross-referencing. HELCOM 1988
baseline corrected 2,800 → 3,500 (Harding & Härkönen 1999). Pre-2000
values flagged as backcasts, not aerial counts. 2014=32,019 exactly
(Lai 2021 HELCOM SEAL 2015). Header citations upgraded across 3 CSVs
with Friedland 2016, Vuorinen BB pooled ranges, Kallio-Nyberg 2009.

## Enabling a WGBAST-comparable Baltic run

```yaml
# configs/example_tornionjoki.yaml (Arc M fixture)
simulation:
  start_date: "2011-04-01"
  end_date: "2016-03-31"
  seed: 42
  population_file: "TornionjokiExample-InitialPopulations.csv"
  adult_arrival_file: "TornionjokiExample-AdultArrivals.csv"
  m74_forcing_csv: "data/wgbast/m74_ysfm_series.csv"   # Arc L

marine:
  zones: [...]
  post_smolt_survival_forcing_csv: "data/wgbast/post_smolt_survival_baltic.csv"  # Arc N
  stock_unit: "sal.27.22-31"
  seal_abundance_csv: "data/helcom/grey_seal_abundance_baltic.csv"   # Arc P
  seal_reference_abundance: 30000.0
  stray_fraction: 0.10   # Arc O

reaches:
  Nemunas:  # shapefile reach, WGBAST-calibrated
    river_name: "Tornionjoki"  # Arc L key
    pspc_smolts_per_year: 880000  # Arc K
    ...
```

Run → emits `smolt_production_by_reach_2015.csv` (WGBAST % PSPC) and
`spawner_origin_matrix_2015.csv` (WGBAST genetic-MSA shape).

## Scientific anchors

All parameter envelopes cross-referenced against peer-reviewed sources:

- **PSPC totals**: WGBAST 2026 §3 (DOI 10.17895/ices.pub.29118545.v3),
  aggregate 3.4M Baltic-wide PSPC
- **M74 life-stage**: Vuorinen 2021 (DOI 10.1080/10236244.2021.1941942),
  yolk-sac freshwater pre-swim-up
- **Smolt age latitudinal gradient**: Skoglund 2024 Paper III
  (DOI 10.54612/a.58aq72nqq6)
- **Post-smolt survival envelope**: ICES 2023 WGBAST §2.5 (3-12%);
  Friedland 2016 (DOI 10.1093/icesjms/fsw178) declining trend
- **Population structure / straying**: Östergren 2021 (DOI 10.1098/rspb.2020.3147);
  Säisä 2005 (DOI 10.1139/f05-094)
- **Grey-seal abundance**: HELCOM core indicator + Westphal 2025
  (DOI 10.1002/aqc.70147) + Carroll 2024 (DOI 10.1111/1365-2656.14065)
- **Counter overdispersion (NB k=50)**: Orell & Erkinaro 2007

Full reference list: `docs/validation/wgbast-roadmap-complete.md` §References.

## Architecture preservation

Every WGBAST feature is **gated behind an opt-in kwarg or CSV path**
that defaults to None/0.0:

| Arc | Default-off knob | Effect when unset |
|-----|------------------|-------------------|
| L | `simulation.m74_forcing_csv = None` | No M74 cull (NetLogo parity) |
| N | `marine.post_smolt_survival_forcing_csv = None` + `current_year = None` | Legacy marine_mort_base |
| O | `marine.stray_fraction = 0.0` | Perfect homing (NetLogo parity) |
| P | `marine.seal_abundance_csv = None` | Legacy static seal_hazard |

Parity tests in `tests/test_run_level_parity.py::TestExampleARunVsNetLogo`
remain stable (the 3 pre-existing Arc H/I residual gaps documented
unchanged across all 8 releases).
