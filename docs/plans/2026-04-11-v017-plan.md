### Task 2.5: Hatchery predator-naivety multiplier on cormorant hazard

**Files:**
- Modify: `src/instream/marine/config.py`
- Modify: `src/instream/marine/survival.py`
- Test: `tests/test_hatchery.py`

> **Scientific grounding (reviewer CRITICAL finding)**: The initial plan
> draft proposed a `hatchery_mortality_multiplier` applied to the
> background hazard. That is scientifically wrong in two ways:
>
> 1. **Mechanism**: the hatchery survival disadvantage is dominated by
>    *predator naivety* during the first 2–4 weeks post-release, not by
>    generic background mortality. Hatchery smolts have been documented
>    to suffer 2–3× higher predation by cormorants and piscivorous fish
>    in this window, then converge on wild rates (Kallio-Nyberg et al.
>    2004, DOI 10.1111/j.0022-1112.2004.00435.x; Jokikokko et al. 2006;
>    Saloniemi et al. 2004 on the Simojoki).
>
> 2. **Duration**: applying a constant 2× multiplier for the whole
>    marine phase compounds far beyond observed lifetime differentials.
>    Over a 365-day ocean year, `(1 - 2×0.001)^365 ≈ 0.69` which
>    already exceeds the observed wild-vs-reared recapture ratio.
>
> **Corrected design**: add `hatchery_predator_naivety_multiplier`
> applied to the **cormorant** hazard only, and only during
> `post_smolt_vulnerability_days`. This matches both the mechanism
> (predation) and the duration (post-release window).
>
> Note: this is separate from `HatcheryStockingConfig.release_shock_survival`
> (Task 2.4), which represents a different concept — immediate
> release-shock survival of the stocked batch itself. The two
> parameters are not redundant.

    def _do_hatchery_stocking(self, current_date):
        """Pop any stocking entries matching *current_date* and inject
        new hatchery fish into dead slots with ``is_hatchery=True``.
        """
        import datetime as _dt
        if not getattr(self, "_hatchery_stocking_queue", None):
            return
        today_iso = current_date.isoformat() if hasattr(current_date, "isoformat") else str(current_date)

        remaining = []
        for entry in self._hatchery_stocking_queue:
            if entry["date"] != today_iso:
                remaining.append(entry)
                continue

            ts = self.trout_state
            dead = np.where(~ts.alive)[0]
            n_add = min(int(entry["num_fish"]), len(dead))
            if n_add <= 0:
                continue
            slots = dead[:n_add]

            sp_idx = entry["species_idx"]
            sp_name = self.species_order[sp_idx]
            sp_cfg = self.config.species[sp_name]

            # Find reach index from name
            reach_idx = self.reach_order.index(entry["reach"]) \
                if entry["reach"] in self.reach_order else 0

            # Random lengths around the mean
            lengths = self.rng.normal(
                entry["length_mean"], entry["length_sd"], size=n_add
            )
            lengths = np.clip(lengths, 5.0, 40.0)
            weights = sp_cfg.weight_A * lengths ** sp_cfg.weight_B

            ts.alive[slots] = True
            ts.species_idx[slots] = sp_idx
            ts.length[slots] = lengths
            ts.weight[slots] = weights
            ts.condition[slots] = 1.0
            ts.age[slots] = 1                  # stocked as yearlings
            ts.reach_idx[slots] = reach_idx
            ts.natal_reach_idx[slots] = reach_idx
            from instream.state.life_stage import LifeStage
            ts.life_history[slots] = int(LifeStage.PARR)
            ts.is_hatchery[slots] = True       # the whole point of this method
            # Reset v0.14.0 marine fields (slot-reuse hygiene, v0.16.0 class bug)
            ts.zone_idx[slots] = -1
            ts.sea_winters[slots] = 0
            ts.smolt_date[slots] = -1
            ts.smolt_readiness[slots] = 0.9    # ready to smoltify (hatchery release)

```python
def apply_post_spawn_kelt_survival(
    trout_state,
    kelt_survival_prob: float,
    min_kelt_condition: float,
    rng,
) -> int:
    """Give post-spawn SPAWNER fish a chance to survive the spawning act
    and out-migrate to the ocean as KELT.

    **Semantics**: ``kelt_survival_prob`` is the **river-exit survival**
    — the probability that a post-spawn SPAWNER survives the spawning act
    and physically reaches the river mouth as a kelt. It is NOT the
    realized repeat-spawning rate. The realized rate emerges from the
    chain:

        realized_repeat = kelt_survival_prob            # survive spawning
                        × P(survive ocean reconditioning)
                        × P(successful second return)

    With the default ``kelt_survival_prob = 0.25`` and the v0.17.0
    calibrated marine hazards, we target a realized repeat rate of
    5-10% which matches the Baltic literature (Niemelä et al. 2004 on
    Teno, Jutila et al. 2006 on Simojoki). The Atlantic-average value
    of ~10-11% (Fleming & Reynolds 2004; Bordeleau et al. 2019) is
    expected for more productive rivers.

    Parameters
    ----------
    trout_state : TroutState
        Fish state arrays. Modified in place: surviving SPAWNERs become
        KELT; the rest keep their SPAWNER stage and are killed by the
        existing post-spawn death block (run immediately after this).
    kelt_survival_prob : float
        Per-fish river-exit survival probability, 0..1.
    min_kelt_condition : float
        Fish with condition below this threshold are ineligible — a
        depleted spawner cannot survive a downstream kelt migration.
    rng : numpy.random.Generator

    Returns
    -------
    int
        Number of SPAWNERs promoted to KELT this call.
    """
    from instream.state.life_stage import LifeStage

    eligible = (
        trout_state.alive
        & (trout_state.life_history == int(LifeStage.SPAWNER))
        & trout_state.spawned_this_season
        & (trout_state.condition >= min_kelt_condition)
    )
    idx = np.where(eligible)[0]
    if len(idx) == 0:
        return 0

    draws = rng.random(len(idx))
    promoted = idx[draws < kelt_survival_prob]
    if len(promoted) == 0:
        return 0

    trout_state.life_history[promoted] = int(LifeStage.KELT)
    # Biological energy depletion: Atlantic salmon lose 40-70% of lipid
    # reserves during spawning (Jonsson et al. 1997; Bordeleau et al. 2019).
    # Halve current condition, then enforce a floor of 0.3 so survivors
    # are still viable but visibly depleted. This is a **multiplicative**
    # reduction, not a `min()` clamp — a min() would be a no-op for
    # already-depleted fish, which is biologically backwards.
    trout_state.condition[promoted] = np.maximum(
        trout_state.condition[promoted] * 0.5,
        0.3,
    )
    return int(len(promoted))
```

### Task 4.1: Write the calibration test first (it will fail)

- [ ] **Step 1: Create the test file**

```python
"""ICES WGBAST end-to-end calibration for Baltic salmon (v0.17.0 Phase 4).

These tests run the full marine-enabled model and assert that the emergent
cohort behaviour lands inside the bands reported by ICES Working Group on
Baltic Salmon and Sea Trout. They exercise the whole pipeline (freshwater
growth -> smoltification -> ocean survival -> return -> post-spawn kelt ->
repeat return) against real-world calibration data.

Reference: ICES WGBAST 2024 report, section on smolt-to-adult survival
(SAR), mean post-smolt length, and repeat-spawner fraction.
"""
from __future__ import annotations
import datetime
from pathlib import Path
import numpy as np
import pytest

CONFIGS = Path(__file__).parent.parent / "configs"
FIXTURES = Path(__file__).parent / "fixtures"


@pytest.mark.slow
class TestICESCalibration:
    @pytest.fixture(scope="class")
    def model(self):
        from instream.model import InSTREAMModel
        from instream.state.life_stage import LifeStage

        start = datetime.date(2011, 4, 1)
        end_date = (start + datetime.timedelta(days=5 * 365)).isoformat()
        m = InSTREAMModel(
            CONFIGS / "example_marine.yaml",
            data_dir=FIXTURES / "example_a",
            end_date_override=end_date,
        )
        # Allocate into dead capacity for a deterministic cohort independent
        # of base-config population size. 3000 PARR gives adequate
        # statistical power to distinguish 5% from 10% SAR at ~80% power
        # (binomial variance on a 3000-fish cohort yields a SAR CI of
        # roughly +/-1.5% at 5% mean). Smaller cohorts flake on rng seed.
        ts = m.trout_state
        dead = np.where(~ts.alive)[0]
        n_parr = min(len(dead), 3000)
        if n_parr < 1500:
            pytest.skip(
                f"TroutState capacity too small for calibration test "
                f"(only {n_parr} dead slots available, need >=1500)"
            )
        parr = dead[:n_parr]
        sp_cfg = m.config.species[m.species_order[0]]

        ts.alive[parr] = True
        ts.species_idx[parr] = 0
        ts.life_history[parr] = int(LifeStage.PARR)
        ts.age[parr] = 1
        ts.length[parr] = 15.0
        ts.weight[parr] = sp_cfg.weight_A * 15.0 ** sp_cfg.weight_B
        ts.condition[parr] = 1.0
        ts.smolt_readiness[parr] = 0.9
        ts.fitness_memory[parr] = 0.5
        ts.reach_idx[parr] = 0           # upstream reach (verify in example_marine.yaml)
        ts.natal_reach_idx[parr] = 0
        ts.zone_idx[parr] = -1           # freshwater
        ts.sea_winters[parr] = 0
        ts.smolt_date[parr] = -1

        m.run()
        return m

    def test_smolt_to_adult_survival_in_ices_band(self, model):
        md = model._marine_domain
        if md.total_smoltified == 0:
            pytest.skip("No smoltifications — cohort collapsed upstream")
        sar = md.total_returned / md.total_smoltified
        # Band justification: ICES WGBAST 2024 cites 2-15% SAR for Baltic
        # wild rivers in recent decades (Gulf of Bothnia ~5-10%, reared
        # ~3-5%). The upper bound is stretched to 18% here to accommodate
        # the iteroparous-return inflation (at 8-10% realized repeat rate
        # the maximum inflation factor is ~1.11×, so 15% × 1.20 ≈ 18%
        # allowing a small stochastic-tail margin on a 3000-fish cohort).
        # The lower bound of 2% is a hard "cohort hasn't collapsed" floor.
        assert 0.02 <= sar <= 0.18, (
            f"Smolt-to-adult return {sar:.3f} outside ICES 2-18% band"
        )

    def test_some_fish_became_kelts(self, model):
        md = model._marine_domain
        # With a 3000-smolt cohort and 0.25 river-exit kelt probability,
        # expect roughly 0.25 × (post-spawn survivors ~ 100-500) ≈ 25-125
        # kelts over the 5-year horizon. Any value > 0 proves the kelt
        # machinery wired correctly; the cohort size makes "> 0" a
        # 5-sigma safe assertion.
        assert md.total_kelts > 0, (
            f"No kelts produced (total_returned={md.total_returned})"
        )

    def test_repeat_spawner_fraction(self, model):
        md = model._marine_domain
        if md.total_returned == 0:
            pytest.skip("No returns in this run")
        repeat_frac = md.total_repeat_spawners / md.total_returned
        # Baltic repeat-spawner rates: Niemelä et al. on Teno ~5-8%,
        # Simojoki virtually nil in some cohorts, Atlantic-average ~10-11%
        # (Fleming & Reynolds 2004). Band 0-15% covers the full range
        # of Baltic rivers plus a small rng tail. The Newfoundland 33%
        # outlier (Campbellton) is NOT plausible for Baltic modelling.
        assert 0.0 <= repeat_frac <= 0.15, (
            f"Repeat-spawner fraction {repeat_frac:.3f} outside "
            f"plausible Baltic 0-15% range"
        )
```
